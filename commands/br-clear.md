## Git Branch Cleanup

원격(origin)에 더 이상 존재하지 않는 로컬 브랜치를 정리합니다.

### Procedure

1. `git fetch --prune`을 실행하여 원격 브랜치 정보를 동기화하고, 삭제된 원격 브랜치의 tracking 참조를 제거한다.
2. 현재 체크아웃된 브랜치와 기본 브랜치(main/master)를 확인한다.
3. 로컬 브랜치 중 원격 tracking 브랜치가 사라진(gone) 브랜치 목록을 조회한다.
   - `git branch -vv`의 출력에서 `[origin/...: gone]`으로 표시되는 브랜치를 필터링한다.
4. 대상 브랜치가 없으면 "정리할 브랜치가 없습니다."를 출력하고 종료한다.
5. 대상 브랜치 목록을 사용자에게 보여주고, **삭제하지 않고 보존할 브랜치**를 선택받는다.
   - 현재 체크아웃된 브랜치는 자동으로 보존 대상에 포함하고 목록에서 제외한다.
   - 사용자가 "전부 삭제"를 선택하면 보존 없이 진행한다.
6. 보존 대상을 제외한 나머지 브랜치를 사용자에게 최종 확인한 후 `git branch -D`로 삭제한다.
7. 삭제 결과를 요약하여 출력한다.

### Rules

- 현재 체크아웃된 브랜치와 기본 브랜치(main/master)는 절대 삭제하지 않는다.
- 삭제 실행 전 반드시 사용자에게 최종 확인을 받는다.
- 삭제 실패 시 원인을 분석하여 사용자에게 알린다.
